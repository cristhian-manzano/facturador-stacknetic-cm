## CRITICAL (must-fix; blocks production)

- [ ] **API ↔ Web detail-endpoint schema mismatch** — `GET /api/v1/invoices/:id` returns a FLAT shape via `toInvoiceDetailWire(row)` (`apps/api/src/invoices/handlers.ts:198`), but `apps/web` binds to `InvoiceDetailSchema` `{invoice, customer, sriDocument, sriEvents}` (the wrapped shape). Detail page will throw schema-mismatch in production. Fix: wrap server response. Add a server-side test that the wrapped shape parses against `InvoiceDetailSchema` from `@facturador/contracts`. (REVIEW-0043 §8 #1, §10 #1). **M**
- [ ] **`apps/api/Dockerfile` does NOT copy `packages/db`** — pre-existing regression flagged when sri-core's Dockerfile was fixed. API runtime image is broken in production. (REVIEW-0020 §9 #8, §11 #1). **S**
- [ ] **Stub mode shipped in api `.env.example` (`SRI_STUB_MODE=true`)** — sri-core refuses to boot in production with stub mode, but the compose `.env.example` defaults make it easy to forget to flip it. Confirm the production deploy explicitly sets `SRI_STUB_MODE=false` and document in a runbook. (REVIEW-0020 §3, §8.6). **S**
- [ ] **Production `__Host-` cookies require HTTPS** — must run behind HTTPS in production for `__Host-facturador_session`/`__Host-facturador_csrf` to round-trip. Deployment-time invariant. (REVIEW-0040 §10, REVIEW-0041 §13). **S** (operational doc) / **L** (deployment infra)
- [ ] **`SRI_CERT_MASTER_KEY_HEX` in env-only / no KMS** — operator with shell access on the host can read `/proc/$pid/environ` and steal the master key that decrypts every tenant's `.p12`. Document KMS rollout (SOPS, AWS KMS, Vault). (REVIEW-0021 §6, §8 follow-up). **L**
- [ ] **No production CSP / reverse-proxy headers configured** — SPEC-0003 §6.5 placeholder; PROMPT-0040 §6 acknowledges. Without CSP, XSS or third-party drive-by exfiltration is unmitigated in prod. (REVIEW-0040 §10). **M**

1. **Lint debt is unresolved across PROMPTs 0006–0030** — ~55 errors live in `apps/api/src/middleware/*`, `test/setup.ts`, `packages/utils/src/audit/*`, `crypto/*`, `errors/*`, `rbac/*`, `service-jwt/*`. CI lint gate is currently disabled or tolerating these. A single dedicated sweep PR is owed. 2. **Bundle size growth on `apps/web`** — login route ships 130 KB gzipped, well over SPEC-0040 NFR-1's 80 KB budget. Every web slice (0040 → 0041 → 0042 → 0043) defers route-level code-splitting to "the next spec". This is now the most pressing UX/perf debt.

3. **Production secret-management is env-only** — `SRI_CERT_MASTER_KEY_HEX`, `SERVICE_JWT_SECRET`, `SESSION_COOKIE_SECRET` all read from `.env`. Operator with shell → reads `/proc/$pid/environ`. KMS rollout is mentioned in REVIEW-0021, REVIEW-0020 §11 #2, REVIEW-0011 §10 (CSP), REVIEW-0010 §10 (Redis). Must land before first real-tenant production deploy.

4. **Polling cadence + horizontal scaling assumptions** — single-replica assumptions in: cert expiry cron (REVIEW-0021 §6 #4), polling job (REVIEW-0026 §10 #2 worker queue follow-up), rate limiter in-memory (REVIEW-0010 §7 R-1). Each needs leader election (Postgres advisory locks) or Redis/BullMQ.

5. **Multi-tenant isolation depends only on app code** — no Postgres RLS yet. SPEC-0004 §6.6 + REVIEW-0004 §6 explicitly defer to a separate spec. Cross-tenant defence today is the `requireTenant` middleware + every `where: { companyId }` clause. A forgotten clause = data leak. Custom ESLint rule deferred to SPEC-0007 §AC-6 (not yet implemented per REVIEW-0004 §7 #3).

6. **API/Web schema drift potential** — the invoice detail mismatch (CRITICAL above) is one concrete instance. The pattern: API ships flat shape, web binds to wrapped shape from `@facturador/contracts`. Without contract-driven server tests (asserting server response parses against the published Zod schema), drift will recur. Add a "contract round-trip" test layer to api.

7. **Async/background jobs ergonomics** — no AsyncLocalStorage helper (REVIEW-0006 §6.4), no worker queue (REVIEW-0026 §10 #2), no job-status surface for the user (REVIEW-0033 §11 #2 async emit). The current synchronous emit handler blocks the user up to 5 s on the orchestrator's P95 (REVIEW-0033 §9 row "Long-running sri-core call"). Cohesive design needed.

8. **Out-of-scope features that block customer onboarding**: RIDE PDF (REVIEW-0043 §10 #2), CSV export/import (REVIEW-0031 #1, REVIEW-0043 #3), anulación electrónica (REVIEW-0033 §11 #5, REVIEW-0043 §10 #4), NC/ND/retencion builders (REVIEW-0023 §12 #4).
